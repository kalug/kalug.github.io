從一個人的 DevOps，到一個 DevOps 的團隊

還記得先前在「敏捷原理與團隊塑造」課堂的最後，講者 William Yeh 提到，難得自己可以待在正在導入敏捷 (Agile) 的團隊，不妨藉由年底的 IT 鐵人賽，把導入 Agile 的種種給寫下來。可計劃總趕不上變化，凍仁便藉由高雄敏捷之旅 (Agile Tour Kaohsiung 2017) 的機會，分享自己近年來的 Agile 和 DevOps 學習旅程。


▲ 12⁄09 於 Agile Tour Kaohsiung 2017 分享的簡報。

由於凍仁是從 Linux 系統工程師 (Linux System Engineer) 起家的，所以 DevOps 的學習路線是從維運 (Opsation)、基礎建設 (Infrastructure)，一路往前推到開發 (Develop)、需求分析 (Request) 和企業投資 (Invest) 等 Agile 探討的相關議題。

在這次的分享中，凍仁先是提到自己拿手 DevOps Tools 的學習經過，再小小提一下正在使用 Scrum 、Kanban 的 DevOps team 是長什麼樣子。

CI/CD/DevOps 既是條不歸路，更是大躍進之路：一旦踏出第一步，就注定你會不斷接觸遠離平庸的新觀點、新技術。

至於背後是不是跑 agile，反而不那麼絕對了。

屢屢從自己與別人演講、教育訓練、Q&A 中歸納出來的心法……
— William Yeh (@william_yeh) 2015年12月20日

現在回頭看 William Yeh 在 2 年前的推文，真的超有感！想當初凍仁參加「Ansible 自動化組態管理實戰講堂」的初衷就是為了少看一本書，沒想到從此想看的書就越來越多了呢！(笑)

在今年參加的付費課程中，最令凍仁印象深刻的，非「《鳳凰項目》沙盤工作坊」莫屬，但願凍仁現在待的團隊，也能早日追上那名為獨角獸 (Unicorn) 的 DevOps team。

相信 2018 年的我們，都一定會更加進步，而接下來的故事，就等明年再繼續說下去吧！

聖誕快樂！
