「敏捷原理與團隊塑造」與「獨角獸計劃」

自從凍仁踏上 DevOps 的偉大航道後，不時會在相關書籍和文獻看到敏捷開發 (Agile) 等字眼，雖有幸能在多年前短暫參與導入 Scrum 的團隊，但一直沒機會見識到導入成功的案例。1

上週六 (06/17) 是凍仁參加 William Yeh 前輩主講《敏捷原理與團隊塑造》課程的日子，也是讓自己解開種種疑惑的日子，雖事前已看過《SCRUM: 用一半的時間做兩倍的事》一書，但對實際面還是抱著很大的疑問。

▲ 課程講義、Agile 科普讀物和隨堂筆記本。

仔細觀察近來的業界生態後就會得知，追求 DevOps 文化的團隊最後都一定會追求 Agile，我想那是因為如果說 DevOps Tools 是各門各派的武功了話，那 Agile 就屬心法了吧！

▲ 在現代 IT 人一定要知道的 Ansible 自動化組態技巧 (3/e) 簡報中，凍仁也曾小小提及 Agile。

這是堂極具互動和軟技術的課程，講者藉由自己的實戰經驗和不少的體驗活動，讓我們意識到 Agile 是什麼？怎麼藉由 Agile 在有限的資源下打出一手漂亮的好牌？

在 Agile 之外，講者更融入了「薩提爾的冰山理論」、「限制理論」、「使用者故事對照」… 等軟技術進行團隊塑造，這都是凍仁未曾接觸過的領域，說講者是本活字典也不為過！

還記得第一次看完《鳳凰項目》2 一書的感動，這是本 DevOps 人的科普讀物，同時也是讓 IT 維運人出現一線曙光的傳奇故事。凍仁想追求書中的「獨角獸計劃」團隊，可只是扮演著布倫特 (Brent) 這個角色是難以達成的，或許當個比爾 (Bill) 一回，自己塑造個團隊會比較快達成！？雖然這次得花上比上次還要久的時間才能消化完課程內容，但至少講者給了凍仁一個明確方向，這就足夠矣！

I have a dream, I want to join the Unicorn project like the Phoenix project book, so I will building by myself ! #ThePhoenixProject #book

留有這份感動的凍仁心想，既然沒法在公司直接導入 Agile，那何不在讓它先融入自己的生活呢？3 相信這個可拋棄式玩具4 一定可以讓凍仁有所領悟的！

▲ 凍仁家的看板 (Kanban)。

最後，就用《SCRUM: 用一半的時間做兩倍的事》一書的第九章標題做結尾吧！

終於看到 #SCRUM用一半的時間做兩倍事 的「第 9 章：改變世界」了，身為瓶子的我，一看到這標題就有種共鳴！ 還令人想起高中跑假服時，學長姊曾說的一句話：「青年改變全世界！」。 #book #翔鷗假期服務隊

我是名自命不凡的系統工程師 (System Engineer)，我想要改變自己、改變團隊並且改變世界！

1 相信不會有人認為站立會議 (Stand-up meeting) 開超過 1、2 小時的團隊會是成功案例。
2 鳳凰項目的原文書名為 The Phoenix Project，可說是入門 DevOps 的必讀書籍之一。
3 想當初自己也是在家中的桌機灌了 Ubuntu 才學好 Linux 的呢！(笑)
4 可拋棄玩具一詞出自《學徒模式》一書，簡而言之就是在一個類似工作環境，但可以隨意玩耍試驗、學習成長，且不會讓自己丟掉工作的玩具。
